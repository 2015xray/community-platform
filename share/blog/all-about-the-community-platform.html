In this Blog post I want to describe the community platform in several ways. Not only do I want to explain the technical details and how you can help contributing to it, I also want to show its possible future, and its historical background. I will also explain a bit of the business decisions behind it.

What is the Community Platform?

The idea for a Community Platform came together with the XMPP server we created for the community. With the XMPP server we had for the first time our own user list of fans and people who like DuckDuckGo. This allowed us to set up a web service on top of this user list, that allows the users to help us translate DuckDuckGo into their language.

What is the upcoming path?

In the next step we still want to concentrate on the translation interface, and add more capabilities to let the user help us manage the translations. We need to be very fast here, because DuckDuckGo is rapidly growing with new features, and we want those features translated into all languages as fast as possible. A very tight coordination with the community, and a notification concept is very relevant here. Also, the voting needs tuning, because people went to vote bad translations, just because other translations were not there. This is not very helpful :). I already started to tune the quality of everything with writing a very detailed manual about our translation infrastructure.

After this, we really need to add more features that integrate users in our process of spreading and extending DuckDuckGo. We want to build a custom plugin management system that allows a more detailed overview of the existing plugin projects and the upcoming projects. It will allow us to have several people, even not coders, help the process of adding more functionality to DuckDuckGo. But also the ability to manage user groups and coordinate the effort of spreading DuckDuckGo.

Also the integration of the so far outsourced Forum into the infrastructure is on the topic list. We want to allow the user to have a unified account to use all the community services of DuckDuckGo.

What is the technology behind the community platform?

One of the primary goals of the community platform is to be 100% open source. We try to write most parts with Perl, because this is the main language used in most of DuckDuckGo, but, of course, we use always the best fitting technology, if there is a wider choice. Our XMPP server is Prosody, which is written in Lua and we use nginx as webserver. The servers are running on Amazon's EC2 services with RightScale Debian images. The Perl web framework we use is the Catalyst Web Framework.

And how can I help?

There are several ways to help. New features for the Community Platform need several levels of engangement, like for example, an idea for the user interface, or the design of special elements, or especially a bit of code, of course :). But it all runs together as part of many ideas. Also, spreading DuckDuckGo and finding ways to help people spread DuckDuckGo is part of the Community Platform, and we will offer more and more features in that area, like user groups, in the near future. I hope we can offer more very easy ways to help in the near future :).
